BLOKUS PROJECT

If you are unfamiliar with the game here are instructions to play by Mattel Games

Although the majority of the programming was done by my teammate Jeremy Eustace, I was heavily involved in planning and structuring the work that went behind it. Including keeping track of all the work that needed to be done by a set date on the progress logs, and communicating with the rest of my team to make sure we were all doing our work on time.
We wanted to make sure that the game was available to play by colour-blind users, so we implemented a colour-blind setting to the game as well.

MY CONTRIBUTIONS TO THE PROJECT

Select Piece Sequence Diagram

The Select Piece sequence diagram follows the functions that are called when a player clicks a piece to play the game. Then the piece of choice is updated in the system and the player places the piece on the board by selecting the 'place a piece' button. The system makes sure the piece is able to fit in the board and updates the board.

Change Color-Blind Setting Sequence Diagram

The Change Colour-Blind Setting sequence diagram shows the sequence of functions that are called when a player selects the colour blind mode and chooses a colour they want to play from the main page of the game. The system then updates the colours of the pieces and the board.

Give Up Fully Dressed Use Case and Sequence Diagram

The Give Up use case follows a success scenario when a player gives up playing the game. When the player chooses to exit the game the system displays a message asking the user if they want to save the current progress of the game. If no is selected the game is wiped and then the player can choose to click the 'exit' button and stop playing the game*.

The Give Up sequence diagram follows the functions called in order to end the game step by step.

Other Contributions

My other contributions involved but was not limited to: creating a vision document in the beginning of the project. Keeping track with the progress log, and planning the next iterations.

Here's a downloadable version of the game we created: Blokus

*Initially the Give Up use case was meant to end the game, however clicking 'give up' button actually skips the players turn and the rest of the players, human/AI keep playing.
